Tab-Scroll + Lazy Session Start — Design

approved 2026-05-18 Pending written-spec review Builds on: aegis-multitab-design
Goal Locked decisions Tab scroll Lazy start Session clock Error handling Testing Non-goals

Goal

Two independent refinements to the multi-tab TUI: (1) the tab bar scrolls sideways so the focused tab is always visible when there are more tabs than fit one line; (2) a tab's harness subprocess is not started until the user's first message in that tab.

Locked decisions

  1. Minimal scroll. On tab switch, scroll only as far as needed to bring the active tab fully into view at the nearest edge; on-screen tabs do not jump.
  2. Lazy start. No session.start() at tab spawn; the subprocess spawns on the first on_input_submitted in that pane.
  3. Session clock from first message. The status-line "session" timer counts from first message (subprocess spawn), not from tab open. An opened-but-unused tab shows 0s.
  4. No new visible state. A pre-message pane shows the normal green ready dot (no distinct "not started" indicator) — it is ready for input.

Tab scroll — src/aegis/tui/widgets.py

TabBar changes from a single Static (one joined markup string) to a HorizontalScroll container holding one private _TabCell(Static) per tab.

  • CSS: height: 1; overflow-x: auto; overflow-y: hidden; scrollbar-size: 0 0; (scrollable, scrollbar hidden, single row).
  • _TabCell renders the existing label markup unchanged: {state.dot} {idx} {handle} [#788C5D]·{slug}·[/#788C5D], a bold * when unseen, wrapped in [reverse] when active.
  • set_tabs(items) keeps the same 6-tuple contract (idx, handle, slug, state, unseen, active) — AegisApp._refresh_tabbar is unchanged. Internally it removes all children and mounts fresh _TabCells (tab count is a handful; reconciliation diffing is needless complexity — YAGNI), tracking the active cell.
  • After (re)mount, call active_cell.scroll_visible(animate=False) — Textual's native "scroll just enough to make it visible" gives the locked minimal behavior with no manual width math. animate=False keeps switching snappy and tests deterministic.
  • Empty list → render a single placeholder cell no tabs (parity with prior behavior).

Lazy start — src/aegis/tui/pane.py

Add self._started = False.

  • on_mount: no longer await self._session.start(). It sets ready (green, no subprocess) and calls refresh_metrics() (renders zeros — see Session clock).
  • on_input_submitted: before the existing empty/working guards' send path, if not self._started: await self._session.start(), self._started = True, self._metrics.begin_session(self._now()). Then the existing turn flow (write user line, state working, start_turn, run_worker) proceeds unchanged.
  • close(): still self.workers.cancel_group(self, "turn") (harmless if no worker), but only await self._session.close() when self._started — never spawn-then-close an unused subprocess.
  • interrupt() is unaffected: it early-returns unless state is working, which is impossible before start.

Session clock — src/aegis/tui/metrics.py

SessionMetrics.session_start becomes float | None (was set at construction).

  • Constructor sets session_start = None (signature stays the same; the now arg becomes ignored/optional — keep it accepted for call-site compatibility but do not anchor on it).
  • New begin_session(now: float) sets session_start = now (idempotent: only sets if currently None).
  • session_seconds(now) returns 0.0 when session_start is None, else now - session_start.
  • Turn metrics (tokens, tools, turn time) are unaffected; pre-message the rendered line reads ↑0 ↓0 · ⚒ 0 · 0s / 0s.

Error handling

  • First-message session.start() failure: the existing _run_turn try/except still covers send/events; wrap the lazy start() so a spawn failure surfaces as the standard red ⚠ harness error note + error dot (not an app crash) and the pane remains closable. _started is set after a successful start() so a failed start can be retried by sending again.
  • scroll_visible before the cell is laid out: call it after mount completes (post-mount), so geometry is known; if the container has zero width yet, Textual no-ops safely and a later refresh re-scrolls.
  • Closing the very last unused tab still exits the app (action_close_tab path unchanged); close() simply skips session.close().

Testing

  • Lazy start (pane pilot, FakeSession): after mount, session.started is False; first submit → started is True and session.start called exactly once; a second message does not re-start; close() on an unstarted pane does not call session.close; on a started pane it does.
  • Session clock (unit + pilot): fresh SessionMetrics → session_seconds(t) is 0.0; after begin_session(t0) → session_seconds(t0+5)==5.0; pre-message StatusBar shows 0s / 0s; begin_session is idempotent.
  • Tab scroll (shell pilot): open enough tabs to overflow a narrow test console; switching to the last tab leaves the active _TabCell within the scroll viewport (assert via the container's scroll offset / region containment) and still rendered with its handle; switching back brings the first tab into view.
  • Regression: the existing shell/pane pilots stay green (set_tabs contract unchanged). The T3-era assertion that read str(TabBar.content) is updated to read the combined text of the _TabCell children.
  • Test work written/validated inline (not delegated); the live claude driver test is unchanged (driver-level).

Non-goals

  • Animated scrolling; visible scrollbar.
  • Click-to-select or drag-reorder tabs.
  • A distinct "not started" tab indicator.
  • Pre-warming / pooling subprocesses; changing the driver protocol.
Deferred consideration

Centered-active-tab scrolling and click-to-focus tabs are natural follow-ups once tabs are individually addressable widgets (this change makes them so) — explicitly out of scope here.
