Goal

Give an agent inside aegis the tools to know its team and pass work to it: discover the other live sessions and the spawnable agent profiles, and hand a packaged context to another live session (fire-and-forget). This is the vision's inter-agent MCP (live context-transfer handoff) — distinct from the blocking subagent spawn (delegate), deferred to slice 3.

Locked decisions

  1. Scope: an AppBridge enabler + aegis_list_sessions + aegis_list_agents + aegis_handoff. aegis_delegate (spawn+block) is slice 3.
  2. Handoff is fire-and-forget: caller composes the context payload; aegis injects it into the target session as a tagged user turn; the call returns immediately. No waiting for a reply (blocking = delegate).
  3. Target busy → clear reject: if the target pane is mid-turn, the tool returns a "target busy, retry" message; no queuing, no interleaving.
  4. Caller identity is self-reported: the per-pane priming tells the agent its own handle; aegis_handoff takes from_handle. Self-handoff / unknown target are validated and rejected.
  5. AppBridge protocol: the server never imports Textual; AegisApp implements the bridge; tools are async and run on the app's loop (safe widget access).

AppBridge — src/aegis/mcp/bridge.py (new)

@dataclass(frozen=True)
class SessionInfo:
    handle: str
    agent_slug: str
    state: str        # AgentState.value: "ready"|"working"|"error"
    active: bool       # is this the focused tab
    unseen: bool

class AppBridge(Protocol):
    def list_sessions(self) -> list[SessionInfo]: ...
    def list_agents(self) -> list[str]: ...
    async def handoff(self, from_handle: str, target_handle: str,
                      context: str) -> str: ...

A pure interface. The server depends only on this; the app supplies a concrete implementation. SessionInfo is the read model the list tool serialises.

Tools — src/aegis/mcp/server.py (extend)

build_server(bridge: AppBridge) -> FastMCP — slice-1's signature gains the bridge. All tools async (run on the app loop):

  • aegis_meta() -> str — briefing (still static) extended to: document aegis_list_sessions / aegis_list_agents / aegis_handoff; state that the agent's own handle is in its system prompt and must be passed as from_handle; explain handoff is a one-way context transfer to a live peer.
  • aegis_list_sessions() -> list[dict] — [bridge.list_sessions()] serialised: handle, agent_slug, state, active, unseen per live tab. Lets an agent see who it can hand off to and whether they're idle.
  • aegis_list_agents() -> list[str] — the configured agent-profile slugs (what could be spawned later). Listing only; spawning is slice 3.
  • aegis_handoff(from_handle: str, target_handle: str, context: str) -> str — awaits bridge.handoff(...); returns its string result (delivered / rejected reason).

Handoff flow

  1. Agent A (knows its handle from the system prompt) calls aegis_handoff(from_handle="A", target_handle="B", context="…summary…").
  2. The tool awaits bridge.handoff → AegisApp.handoff: resolve target pane by handle among self._panes.
    • target_handle == from_handle → return "handoff rejected: cannot hand off to yourself".
    • no pane with target_handle → return "handoff rejected: no session 'B' (use aegis_list_sessions)".
    • target pane.state is AgentState.working → return "handoff rejected: 'B' is busy, retry shortly".
  3. Otherwise await target.deliver_handoff(from_handle, context) and return "delivered to B".

ConversationPane.deliver_handoff(from_handle, context)

Reuses the normal submission path. Factor the body of on_input_submitted (write user line → set working → start_turn → run_worker(_run_turn(text))) into a private _submit(text); on_input_submitted and deliver_handoff both call it. The handoff's injected line is rendered with the user style but prefixed [handoff from <from_handle>] so the receiving agent (and the human watching that tab) sees it is an inter-agent transfer, then the context. The target's turn runs normally; A does not wait.

Caller identity (self-report)

The MCP server is shared; a tool call carries no intrinsic pane identity. aegis bakes the handle into the per-pane system prompt: PRIMING becomes a template with a {handle} slot; build_argv(agent, cwd, mcp_url, handle) formats it (the same additive thread slice-1 used for mcp_url). The agent reads its handle from its system prompt and passes it as from_handle. Trust-based — acceptable for single-user personal infra; the validation (self/unknown/busy) bounds the blast radius. Enforced per-pane identity is a future hardening (recorded, not this slice).

Wiring (delta to slice-1)

  • build_server(bridge) — slice-1's no-arg build_server() gains the bridge param.
  • AegisMCP(bridge) — slice-1's runtime gains a bridge it forwards to build_server.
  • AegisApp implements AppBridge (list_sessions/list_agents/handoff methods on the app) and binds itself as the bridge before mcp.start() in on_mount (e.g. self._mcp.bind(self), or it constructs AegisMCP(self) there). Tests still inject FakeMCP (no bridge needed); server-tool tests use a FakeBridge.
  • make_session(profile, mcp_url, handle) → session(agent, cwd, mcp_url, handle) → build_argv(..., handle); _spawn passes handle (it already generates it).
  • ConversationPane: extract _submit(text); add async deliver_handoff(from_handle, context).

Error handling

  • All handoff failure modes return a plain string the calling agent can act on (reject reasons above) — not exceptions across the MCP boundary.
  • list_sessions/list_agents never fail: empty list if (impossibly) no panes; they are pure reads of app state.
  • A handoff whose target errors mid-delivery: deliver_handoff only enqueues the turn (same as a user submit) — any harness error surfaces in the target pane's transcript, not the caller's tool result (which already returned "delivered").
  • Tools run on the app loop; they must not block it — they only read small state or call an async pane method that itself spawns a worker (non-blocking), so a tool call returns promptly.
  • Bridge unbound (defensive): if AegisMCP somehow starts without a bound bridge, tools return "aegis bridge unavailable" rather than crash. The wiring guarantees binding before start; this is belt-and-suspenders.

Verification

  • unit (FakeBridge) — aegis_list_sessions serialises SessionInfo fields; aegis_list_agents returns the slugs; aegis_handoff returns the delivered string, and the three reject strings for self / unknown / busy.
  • unit — aegis_meta() briefing now names the three new tools and the "handle is in your system prompt" instruction; PRIMING.format(handle=...) embeds the handle; build_argv(..., handle) system-prompt arg contains it.
  • pilot (real AppBridge) — an AegisApp with two panes: app.list_sessions() reflects both handles/states/active; await app.handoff(a, b, "ctx") injects [handoff from a] into pane b's transcript and sets b working; self/unknown/busy each return the right reject and do not modify any pane.
  • pilot — deliver_handoff and on_input_submitted share _submit (a normal user message and a handoff both produce a working turn; existing submit tests still pass).
  • live extension (slow, skips w/o claude) — an agent is told to call aegis_list_sessions; assert the stream shows the tool call and its own handle in the result. (Two-agent live handoff is exercised by the pilot, not the slow live test.)
  • Hermetic suite green; FakeMCP path unchanged. Test work written/validated inline.

Non-goals

  • aegis_delegate — spawn a subagent into a new session and block on its result (slice 3).
  • Blocking / round-trip handoff; awaiting the target's reply.
  • Queuing handoffs to a busy target (reject now; queue is a later refinement).
  • Enforced per-pane MCP identity (self-report this slice).
  • Cross-instance / multi-host; observing another agent's stream.
Deferred / next

Slice 3: aegis_delegate (spawn-into-session + block + result capture). Later: enforced identity, busy-queue, "ask" (a bounded blocking question to a peer distinct from full delegate), observe/attach.
